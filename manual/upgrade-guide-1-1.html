CGLocalization

From the version 1.1 and on, the .dll files were removed from the package and the source code files were included.

When upgrading from .dll to source code, the references to components inside the CGLocalization.dll file are lost. Due to this issue we must to update these lost references manually.
The lost references are the LocalizationAsset and ChangeLanguage components.

Fixing Missing Script in your LocalizationAsset

  1. Select Edit > Project Settings > Editor and set:
        Version Control Mode = Visible Meta Files
        Asset Serialization Mode = Force Text
  2. Go to the folder where your LocalizationAsset is located in you OS File Explorer and open it in a text editor.
  3. Select Assets > Create > Localization to create a new LocalizationAsset in the same location of your asset.
  4. Open the new created LocalizationAsset in the same text editor.
  5. Copy the m_Script: {fileID: XXX, guid: XXX, type: 3} line in the new created LocalizationAsset and replace the same line in your LocalizationAsset.
  6. Now when you switch to Unity your LocalizationAsset is restored.
  7. Delete the new created LocalizationAsset because is no longer needed.

Now you have to reassign your LocalizationAsset in the LocalizationSettings

  1. Select Window > CGLocalization > Editor to open the CGLocalization window.
  2. Click the Localization Settings button at the right in the toolbar of the window.
  3. Assign your LocalizationAsset in the Localization Asset property in the popup window that appears.

Fixing Missing Script for ChangeLanguage

If you have been used the ChangeLanguage script to allow the end user of your project to change the language, you must to reassign the references manually.

When you play a scene, in the Console window will appear warnings like this one:
The referenced script (Unknown) on this Behaviour is missing!

Just click a warning and then reassign the script.
